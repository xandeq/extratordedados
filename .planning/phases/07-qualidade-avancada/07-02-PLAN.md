<objective>
Create the crm_sent_leads cache table and wire it into sync_lead_to_alexandrequeiroz() to prevent re-sending leads that are already in the CRM.

Purpose: The current sync checks the CRM API via GET for every lead on every sync run. The cache table short-circuits this: if the email was sent before, skip immediately — no network call needed.

Output:
- SQL DDL in db_alter_leads.sql: CREATE TABLE crm_sent_leads + 2 indexes
- Cache READ at the top of sync_lead_to_alexandrequeiroz(): check local table before API call
- Cache WRITE after successful CRM sync: insert email/phone/whatsapp/crm_id into table
- Run migration SQL on the VPS PostgreSQL container as part of deploy
</objective>

-- ── Phase 7 QUAL-04: CRM dedup cache ────────────────────────────────────────
-- Cache table to track leads already sent to the external CRM.
-- Prevents re-sending the same lead across multiple sync runs.
-- Email comparison is case-insensitive (LOWER index).
-- No FK to leads table — cache is global, not per-batch.
CREATE TABLE IF NOT EXISTS crm_sent_leads (
    id          BIGSERIAL PRIMARY KEY,
    email       TEXT NOT NULL,
    phone       TEXT,
    whatsapp    TEXT,
    crm_id      TEXT,
    sent_at     TIMESTAMP DEFAULT NOW()
);

CREATE UNIQUE INDEX IF NOT EXISTS idx_crm_sent_leads_email
    ON crm_sent_leads (LOWER(email));

CREATE INDEX IF NOT EXISTS idx_crm_sent_leads_sent_at
    ON crm_sent_leads (sent_at DESC);
-- ── End Phase 7 QUAL-04 ──────────────────────────────────────────────────────
