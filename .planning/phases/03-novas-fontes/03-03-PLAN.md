### Task 1 — AWS SM key + `_get_prospeo_key()` + `enrich_linkedin_prospeo()`

**Files:**
- `app/backend/app.py` (modify)
- `app/backend/requirements.txt` (modify — no new package needed, uses `requests` already imported)

**Action:**

**Step 1 — Create AWS SM secret:**
```bash
python -m awscli secretsmanager create-secret \
  --name "tools/prospeo" \
  --secret-string '{"PROSPEO_API_KEY":""}' \
  --description "Prospeo Social URL API — 75 credits/month free, LinkedIn URL to email"
```

**Step 2 — Add `_get_prospeo_key()` to app.py:**
Add immediately after `_get_outscraper_key()`. Same pattern:

```python
def _get_prospeo_key():
    """Fetch Prospeo API key from AWS SM (with env fallback)."""
    key = os.environ.get('PROSPEO_API_KEY')
    if not key:
        try:
            import boto3
            sm = boto3.client('secretsmanager', region_name='us-east-1')
            secret = sm.get_secret_value(SecretId='tools/prospeo')
            key = json.loads(secret['SecretString']).get('PROSPEO_API_KEY')
        except Exception as e:
            print(f"[prospeo] AWS SM error: {e}")
    if not key:
        raise ConfigError("[PROSPEO] API key não disponível — crie conta em prospeo.io e salve em AWS SM tools/prospeo")
    return key
```

**Step 3 — Add `enrich_linkedin_prospeo(linkedin_url)` to app.py:**
Add after `_get_prospeo_key()`. This function enriches a single LinkedIn profile URL to find their email.

Implementation:
- Validates `linkedin_url` starts with `https://www.linkedin.com/in/` or `https://linkedin.com/in/`
- Calls `POST https://api.prospeo.io/social-url-enrichment` with `X-KEY: {api_key}` header and `{"url": linkedin_url}` body, timeout=10s
- Uses `http_requests.post()` (same module alias already in app.py for `requests`)
- On success (200): returns `{"email": email_value, "email_type": type, "email_status": status}` or `{}` if `email.value` missing
- On 402 / `"credits"` in response text / `"quota"` in response text: raises `ConfigError("[PROSPEO] Créditos esgotados")` — caller handles gracefully
- On any other error: returns `{}` (never raises)
- Logs: `print(f"[prospeo] {linkedin_url} → {email or 'not found'}")`

**Verify:** `grep -n "def enrich_linkedin_prospeo\|def _get_prospeo_key" app/backend/app.py` returns two matches.

**Done:** `_get_prospeo_key()` and `enrich_linkedin_prospeo()` exist in app.py. Never raises except ConfigError on quota.

### Task 2 — `POST /api/leads/<id>/enrich-linkedin` endpoint + LinkedIn hook

**Files:**
- `app/backend/app.py` (modify)

**Action:**

**Part A — New endpoint `POST /api/leads/<id>/enrich-linkedin`:**

Add a new Flask route after the existing `POST /api/leads/<id>/verify-email` endpoint (search for `verify-email` to find the location). Rate limit: `@limiter.limit("30/hour")`.

Logic:
1. `verify_token()` → 401 if not authenticated
2. Fetch lead from DB by `id` → 404 if not found
3. Check `lead['linkedin']` is non-empty → 400 with `{"error": "Lead sem LinkedIn URL"}` if empty
4. Check `_get_prospeo_key()` → if raises ConfigError, return 503 `{"error": "Prospeo não configurado"}`
5. Call `enrich_linkedin_prospeo(lead['linkedin'])` → if returns `{}`, return 200 `{"enriched": false, "message": "Email não encontrado"}`
6. If email found: `UPDATE leads SET email=?, last_verified_at=NOW() WHERE id=?` — only if `lead['email']` is currently NULL or empty
7. Return `{"enriched": true, "email": email_value, "email_type": type, "email_status": status}`
8. Handle ConfigError (quota exceeded): return 429 `{"error": "Créditos Prospeo esgotados este mês"}`

**Part B — Hook into `process_linkedin_massive()`:**

In `process_linkedin_massive()` (around line ~11842), after `_save_leads_to_batch()` saves each lead, add a credit-guarded Prospeo enrichment call. Add this AFTER the lead save, not before:

```python
# Prospeo enrichment — only if lead has linkedin URL but no email
# Guard: max 75 leads per month (matches Prospeo free tier)
if not lead.get('email') and lead.get('linkedin') and prospeo_credits_used < 75:
    try:
        enriched = enrich_linkedin_prospeo(lead['linkedin'])
        if enriched.get('email'):
            # Update the just-inserted lead's email in DB
            try:
                c.execute(
                    "UPDATE leads SET email=%s, last_verified_at=NOW() WHERE batch_id=%s AND linkedin=%s AND email IS NULL",
                    (enriched['email'], batch_id, lead['linkedin'])
                )
                prospeo_credits_used += 1
            except Exception:
                pass
    except ConfigError:
        print("[prospeo] Quota esgotada — parando enrichment para este job")
        prospeo_credits_used = 75  # stop further calls
```

Initialize `prospeo_credits_used = 0` at the start of `process_linkedin_massive()`, before the jobs loop. This is a per-run counter (not persisted). It limits waste of free credits in a single massive search run.
