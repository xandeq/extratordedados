### Task 1 — `cnpj_rf` table migration in `init_db()`

**Files:**
- `app/backend/app.py` (modify)

**Action:**

Inside `init_db()` in `app/backend/app.py`, add a Phase 3 migration block immediately after the Phase 2 dedup/global index block (around line 1767). Follow the exact same pattern already used for Phase 1 and Phase 2 migrations — `CREATE TABLE IF NOT EXISTS`, `CREATE INDEX IF NOT EXISTS`, wrapped in try/except with `conn.rollback()` on error.

Add this block:

```python
# Phase 3: cnpj_rf table — Receita Federal local lookup
try:
    c.execute('''
        CREATE TABLE IF NOT EXISTS cnpj_rf (
            cnpj            CHAR(14) PRIMARY KEY,
            razao_social    TEXT,
            nome_fantasia   TEXT,
            situacao        SMALLINT,
            cnae_principal  VARCHAR(7),
            logradouro      TEXT,
            numero          VARCHAR(10),
            complemento     TEXT,
            bairro          TEXT,
            cep             VARCHAR(8),
            municipio_cod   INTEGER,
            uf              CHAR(2),
            ddd1            VARCHAR(3),
            telefone1       VARCHAR(9),
            ddd2            VARCHAR(3),
            telefone2       VARCHAR(9),
            email           TEXT,
            data_abertura   DATE,
            porte           SMALLINT,
            matriz_filial   SMALLINT
        )
    ''')
    # Partial indexes on active companies only (situacao=2 means ativa)
    c.execute('''
        CREATE INDEX IF NOT EXISTS idx_cnpj_rf_uf_municipio
          ON cnpj_rf (uf, municipio_cod) WHERE situacao = 2
    ''')
    c.execute('''
        CREATE INDEX IF NOT EXISTS idx_cnpj_rf_cnae
          ON cnpj_rf (cnae_principal) WHERE situacao = 2
    ''')
    print("[init_db] Phase 3: cnpj_rf table ready")
except Exception as e:
    print(f"[init_db] Phase 3 cnpj_rf: {e}")
    conn.rollback()
```

### Task 2 — `enrich_from_rf_local()` + `enrich_cnpj_with_fallback()` in app.py

**Files:**
- `app/backend/app.py` (modify)

**Action:**

Add two functions to `app/backend/app.py` immediately after the existing `enrich_cnpj_brasilapi()` function (after line ~1560). Do not remove `enrich_cnpj_brasilapi()` — it becomes one step in the fallback chain.

**Function 1: `enrich_from_rf_local(cnpj_raw)`**

Direct psycopg2 lookup against `cnpj_rf` table. Uses `DB_CONFIG` already defined in the module. Normalizes CNPJ to 14 digits by stripping non-digits (`re.sub(r'\D', '', ...)`). Returns normalized dict (same schema as `enrich_cnpj_brasilapi` output) or `{}` on miss. Target latency: < 10ms. Must never raise — all exceptions caught and logged with `print(f"[rf_local] ...")`.

Return dict keys to match the existing `enrich_cnpj_brasilapi` schema: `razao_social`, `nome_fantasia`, `phone`, `city`, `state`, `cep`, `address`, `cnae_code`, `situacao`, `porte`, `email_rf`, `source`. Set `source='rf_local'`.

Situacao mapping: `{2: 'ativa', 3: 'suspensa', 4: 'inapta', 8: 'baixada'}`.

Phone formatting: if `ddd1` and `telefone1` both exist, return `ddd1 + telefone1` as phone string.

**Function 2: `enrich_cnpj_with_fallback(cnpj_raw)`**

5-level fallback chain. Returns on first success. Each level has its own timeout. Logs which level succeeded with `print(f"[cnpj_fallback] hit level N: {source}")`.

```
Level 1: enrich_from_rf_local(cnpj)         — SQL, no network, 3s timeout via threading
Level 2: http://localhost:3000/{cnpj_fmt}    — Minha Receita local (may not be running), 3s timeout
Level 3: enrich_cnpj_brasilapi(cnpj)         — existing function, 8s timeout (already built in)
Level 4: receitaws.com.br/v1/cnpj/{cnpj}    — 3 req/min free, 8s timeout
Level 5: publica.cnpj.ws/cnpj/{cnpj}        — already in use elsewhere, 8s timeout
```

For levels 2, 4, 5: make plain `http_requests.get()` calls (same as `enrich_cnpj_brasilapi`). Level 2 must silently skip if connection refused (Minha Receita not yet running is normal). Normalize the response from each level to the same dict schema before returning.

**Wire into existing endpoint:** In `POST /api/leads/enrich-cnpj` handler (search for `enrich_cnpj_brasilapi(cnpj)` at line ~8003), replace the direct call with `enrich_cnpj_with_fallback(cnpj)`. No other change to the endpoint.

### Task 3 — Standalone import script + RF docs

**Files:**
- `scripts/import/import_receita_federal.py` (create)
- `docs/RECEITA_FEDERAL_IMPORT.md` (create)

**Action for `scripts/import/import_receita_federal.py`:**

Standalone script that runs **on the VPS** (not inside Flask). Connects to PostgreSQL via psycopg2 using credentials fetched from AWS SM `extratordedados/prod` (with `.deploy.env` fallback, same pattern as `deploy.py`). No Flask imports.

Key implementation details:
- `RF_MIRROR_BASE = "https://arquivos.receitafederal.gov.br/cnpj/dados_abertos_cnpj/"` — script discovers latest month directory by scraping the index page (fallback: hardcode current month `2026-02` if discovery fails)
- Downloads `Estabelecimentos{N}.zip` files (there are multiple shards, e.g., `Estabelecimentos0.zip` through `Estabelecimentos9.zip`) and `Empresas{N}.zip` files
- Checks disk space before starting: if `shutil.disk_usage('/').free < 30 * 1024**3` (30 GB), prints warning and exits with code 1
- Opens CSVs with `encoding='latin-1', errors='replace'`, delimiter `';'`
- Concatenates `row[0] + row[1] + row[2]` (cnpj_basico 8 + cnpj_ordem 4 + cnpj_dv 2) = 14-digit CNPJ
- Filters `ONLY_ACTIVE = True` (default): only inserts rows where `row[4] == '02'` (situacao_cadastral = ativa)
- Batch inserts with `BATCH_SIZE = 10_000` using `executemany` + `ON CONFLICT (cnpj) DO NOTHING` (safe for re-runs)
- Logs progress every 100,000 rows: `print(f"[import] {total:,} linhas processadas, {inserted:,} inseridas...")`
- Deletes each ZIP after successful parse to recover disk space
- Run duration estimate in logs: records start time, prints elapsed at end
- `nohup`-safe: all output goes to stdout (redirectable with `> /tmp/rf_import.log`)

Column mapping from ESTABELECIMENTOS CSV (validate against metadata PDF before first run):
```
row[0]  cnpj_basico
row[1]  cnpj_ordem
row[2]  cnpj_dv
row[4]  situacao_cadastral
row[6]  nome_fantasia
row[11] cnae_fiscal_principal
row[15] logradouro
row[16] numero
row[17] complemento
row[18] bairro
row[19] cep
row[20] uf (sigla 2 chars)
row[21] municipio (codigo RF integer)
row[22] ddd_1
row[23] telefone_1
row[24] ddd_2
row[25] telefone_2
row[27] email
```

Include a `--dry-run` flag: parses first 1,000 rows and prints sample data without inserting. Useful for validating column indexes before full run.

For `Empresas` files, join on `cnpj_basico` to enrich `razao_social`, `data_abertura`, `porte`, `matriz_filial`. This requires a two-pass approach or an in-memory dict for small batches.
